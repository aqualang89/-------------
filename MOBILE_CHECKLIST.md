# Чеклист мобильной адаптации — Рипарий

**Правило:** перед каждым коммитом проходить этот чеклист. Если хотя бы один пункт не проходит — коммита нет, чиним.

---

## 1. Тестовые устройства (DevTools → Toggle Device Toolbar)

| Устройство | Ширина | Что проверяем |
|---|---|---|
| iPhone SE | 375px | Самый узкий экран, всё должно влезать |
| iPhone 14 Pro Max | 430px | Средний современный телефон |
| iPad Mini | 768px | Планшет, переходная зона |
| iPad Pro / Desktop | 1024px+ | Планшет в landscape / десктоп |

**Как открыть:** F12 → `Ctrl+Shift+M` (или кнопка «устройства») → выбрать устройство → обновить страницу.

---

## 2. Чеклист на каждом устройстве

### Визуал
- [ ] Нет горизонтальной прокрутки (проверить: тянуть пальцем/мышью вправо — страница не двигается)
- [ ] Нет обрезанного текста (все заголовки, кнопки, цены видны целиком)
- [ ] Кнопки и ссылки не меньше 44×44px (минимум для пальца)
- [ ] Отступы слева/справа минимум 16px (лучше 20-24px), не прилипают к краям
- [ ] Шрифт читаемый — не мельче 14px для основного текста, 12px только для мелких меток
- [ ] Изображения не растянуты и не сплющены (`object-fit: cover` где нужно)

### Навигация
- [ ] Бургер-меню открывается и закрывается
- [ ] Пункты меню тапаются пальцем (не слишком близко друг к другу)
- [ ] Меню не выходит за экран
- [ ] При скролле меню не дёргается

### Модалки и виджеты (чат, корзина)
- [ ] Чат-виджет не перекрывает контент (можно закрыть/свернуть)
- [ ] На мобилке модалки на весь экран или центрированы с отступами
- [ ] Поля ввода в модалках — тап по полю открывает клавиатуру, поле не уезжает за экран
- [ ] Кнопка «закрыть» в модалке видна и тапается

### Скролл и поведение
- [ ] Страница скроллится плавно, нет «залипания»
- [ ] Кнопка «назад» в браузере — страница возвращается в нормальное положение (не съезжает вверх/вниз)
- [ ] При открытии клавиатуры (поиск, чат) контент не уезжает за пределы экрана
- [ ] Нет `overflow: hidden` на `html` или `body` (ломает скролл на iOS)

### Перформанс
- [ ] Анимации плавные, нет лагов на слабом устройстве (iPhone SE в DevTools)
- [ ] Изображения не грузятся в 4K на мобилке (Responsive Images или `srcset`)

## 3. CSS-правила (mobile-first)

### Правильно:
```css
/* База — мобилка */
.card {
  width: 100%;
  padding: 16px;
}

/* Планшет */
@media (min-width: 768px) {
  .card {
    width: 48%;
  }
}

/* Десктоп */
@media (min-width: 1024px) {
  .card {
    width: 31%;
  }
}
```

### Неправильно (desktop-first):
```css
/* Так НЕ пишем — это текущая проблема */
.card {
  width: 31%;      /* десктоп */
}
@media (max-width: 768px) {
  .card {
    width: 100%;   /* «перебиваем» для мобилки */
  }
}
```

### Запрещено:
- `position: absolute` для layout (только для точного позиционирования иконок/меток)
- Фиксированные `height` (использовать `min-height`)
- `!important` в адаптивных стилях
- Хардкод пикселей в media queries (только CSS-переменные)
- `overflow: hidden` на `html`/`body`
- `touch-action: none` на всей странице

## 4. Breakpoints проекта (единые)

Использовать ТОЛЬКО эти значения, никаких других:

```css
/* В variables.css или :root */
--bp-mobile: 0px;
--bp-tablet: 768px;
--bp-desktop: 1024px;
--bp-wide: 1440px;
```

```css
/* Пример использования */
@media (min-width: 768px) { /* планшет */ }
@media (min-width: 1024px) { /* десктоп */ }
@media (min-width: 1440px) { /* широкий экран */ }
```

## 5. Как проверить «назад»

1. Открыть страницу
2. Проскроллить вниз
3. Перейти по любой ссылке
4. Нажать «назад» в браузере
5. **Ожидаемо:** страница на том же скролле, ничего не съехало

Если съехало — проверить:
- Нет ли `scroll-behavior: smooth` в глобальных стилях (ломает `history.scrollRestoration`)
- Нет ли `position: fixed` элементов, которые меняют высоту страницы
- В Nuxt: `nuxt.config.ts` → `app.pageTransition = false` или настроить корректно

---

## 6. iOS-специфика

- `100vh` на iOS = ложная высота (учитывает панель Safari). Использовать `100dvh` или `-webkit-fill-available`
- `position: fixed` + клавиатура = элемент может уехать. Для модалок использовать `position: fixed` с `top: 0; left: 0; right: 0; bottom: 0` и скролл внутри
- `-webkit-tap-highlight-color: transparent` — убрать синюю вспышку при тапе
- `input[type="text"]` — шрифт не меньше 16px, иначе iOS зумает при фокусе

## 8. Памятка: почему mobile-first

| Desktop-first | Mobile-first |
|---|---|
| Пишем сложный код, потом «ломаем» его для мобилки | Пишем простой код, потом «добавляем» для десктопа |
| Много `@media (max-width)` с переопределениями | Мало media-запросов, только расширение |
| Легко забыть про мобилку | Мобилка — по умолчанию, не забыть невозможно |
| Каскад CSS становится запутанным | Каскад предсказуемый |
