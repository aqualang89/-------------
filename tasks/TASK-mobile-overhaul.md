## Цель

Системно починить мобильную версию сайта Рипарий по трём проблемам:
1. **Горизонтальный сдвиг (overflow-x)** на мобиле — экран периодически "съезжает вправо", особенно после back-навигации. Видна тонкая полоска справа.
2. **Навигация / бургер** — баги поведения: бургер слева вместо справа после открытия меню; бургер не пропадает при скролле вверх; меню остаётся открытым при смене страницы; на узких телефонах (≤375px) бургер обрезается справа.
3. **Мёртвый код** в стилях — есть устаревшие правила (`.top-nav`) которые могут конфликтовать с актуальными.

После задачи мобильная версия должна работать **предсказуемо во всех сценариях**: первый заход, back-навигация, скролл вниз/вверх, открытие/закрытие меню, переход на внутреннюю страницу и обратно.

## Контекст / зачем

Макс показал два бага в продакшене:
1. После нажатия "Назад" со страницы услуг — на iPhone виден ~1-2px белой полоски справа, страница чуть-чуть скроллится вправо
2. Открыл бургер на мобиле — закрытое меню зависает слева, бургер исчезает или дублируется

Корневая причина: смешанные desktop-first и mobile-first стили, реактивный `compact` state срабатывает на мобиле бессмысленно, `100vw` без учёта скроллбара.

## Что менять

### 1. Найти и убить все источники horizontal overflow

**Системно:** пройди по всем CSS-файлам проекта (`assets/css/main.css`, scoped стили в `pages/*.vue` и `layouts/*.vue`, `components/*.vue`), найди:
- `100vw` — заменить на `100%` везде где это работает (для `.sh-mobile-menu` и подобных fixed-элементов на всю ширину). Если действительно нужна полная ширина — использовать `left: 0; right: 0` без `width`. Если нужна именно "viewport минус scrollbar" — посчитать через JS или использовать `100dvw` (поддерживается современными браузерами).
- `position: fixed` без `max-width` — проверить что элемент не выходит за правый край на узких экранах
- `transform: translateX(...)` — `.sh-horizontal-track` использует это для скролла. При SPA back-nav transform может остаться > 0 → горизонтальный overflow. Добавь страховку: `body { overflow-x: clip }` или `html { overflow-x: clip }`. `clip` лучше чем `hidden` — не создаёт scroll context, не ломает `position: sticky`.

**Конкретные точки:**
- `assets/css/main.css:557` — `width: calc(100vw - 16px)` для `#quiz-box` → заменить на `left: 8px; right: 8px; width: auto`
- `assets/css/main.css:28-30` — `body { overflow-x: hidden }` → проверь не ломает ли sticky. Если ломает — заменить на `overflow-x: clip` или вынести на `html`. По STATE.md от 2026-05-07 `overflow-x: hidden` на html сломал sticky. Используй `clip` или оставь `hidden` на `body` + проверь sticky.
- Любые места где элемент имеет `width: 100vw` или `min-width: 100vw` — убрать.

### 2. Убить мёртвый код `.top-nav`

В `assets/css/main.css` строки **38–93** (примерно) — селекторы `.top-nav`, `.nav-inner`, `.nav-logo-text`, `.nav-links`, `.nav-link`, `.nav-cta`. Проверь grep'ом по всему проекту — используется ли где-то класс `.top-nav` или `.nav-inner`? Если **нет** — удалить весь блок целиком.


Если в результате только определения CSS — это мёртвый код, удалить.

### 3. Рефакторинг навигации в `layouts/default.vue`

Текущее поведение (проблемное):
- `compact` срабатывает при `window.scrollY > 80` **на всех экранах**, включая мобайл — на мобиле это бессмысленно (десктоп-ссылки скрыты, бургер всегда виден)
- `menuOpen` не сбрасывается при смене страницы и скролле
- На мобиле desktop-compact-стили мобильного меню (`right: 36px; width: 220px`) могут просочиться через каскад

**Что сделать:**

#### 3.1. `compact` только для десктопа

```js
function onScroll () {
  // на мобиле compact не нужен — там бургер всегда видим, ссылок нет
  if (window.innerWidth < 768) {
    compact.value = false
    return
  }
  if (!scrollTick) {
    requestAnimationFrame(() => {
      compact.value = window.scrollY > 80
      scrollTick = false
    })
    scrollTick = true
  }
}
```

Также пересчитать compact при resize (если юзер повернул телефон / открыл DevTools).

#### 3.2. `menuOpen` закрывается при смене страницы

В `<script setup>`:
```js
const route = useRoute()
watch(() => route.path, () => {
  menuOpen.value = false
})
```

#### 3.3. На мобиле бургер всегда справа, никогда не пропадает

В CSS `@media (max-width: 767px)` (или `@media (max-width: 768px)` если используешь существующий стиль — единый формат):
- `.sh-nav-burger { opacity: 1 !important; pointer-events: auto; transform: none; position: relative; right: 0; }` — НО только если без `!important` нельзя обойтись (избегай `!important` если можно). Лучший путь — поднять специфичность чем-то типа `.sh-nav .sh-nav-burger` или поместить в правильном порядке каскада.
- Убедиться что `.sh-nav-inner { justify-content: flex-end }` работает.

#### 3.4. Мобильное меню — НЕ накрывает бургер

`.sh-mobile-menu.open` имеет `z-index: 1001`. Сам `.sh-nav { z-index: 1000 }`. Когда меню открыто на мобиле — оно может перекрыть бургер. Это **некорректно** — пользователь должен видеть бургер чтобы закрыть меню.

**Решение:**
- На мобиле `.sh-mobile-menu` должно начинаться **под** навигацией (`top: 72px` — уже так), не на 0
- `.sh-nav` z-index выше или равен `.sh-mobile-menu`. Сделать `.sh-nav { z-index: 1002 }` или `.sh-mobile-menu { z-index: 999 }` — лишь бы бургер был НАД меню.
- Когда меню открыто, бургер превращается в крестик (`<span>` rotate 45deg) — это правильно по UX (видишь крестик → знаешь как закрыть).

## Чего НЕ трогать

- HTML структуру навигации (`<nav class="sh-nav">`, `<button class="sh-nav-burger">`, `<div class="sh-mobile-menu">`)
- Логику `useScrollReveal` / `data-reveal` атрибуты
- Hero, аквариум, ripples, интро — это всё работает
- Вертикальные карточки галереи (`.sh-hcard*`) — это вчерашняя работа, не трогаем
- Каталог товаров, корзину, чат-виджет — функционально не меняем
- `STATE.md`, `DESIGN.md`, `MOBILE_CHECKLIST.md`, `AGENTS.md` — это документация, в коде не править

## Edge cases / грабли

- **`overflow: hidden` vs `clip`:** `clip` лучше — не создаёт scroll-context и не ломает `position: sticky`. На современных браузерах поддержка есть. Если не уверен — оставь `hidden` на `body`, проверь sticky.
- **resize event:** добавь debounce 100мс на resize handler, не дёргай compact 60 раз в секунду при ресайзе.
- **iOS Safari address bar:** при скролле панель Safari убирается, `100vh` меняется — НЕ использовать `100vh`, использовать `100dvh`.
- **Sticky после ремонта overflow:** проверь что галерея на главной (`.sh-horizontal-sticky`) всё ещё работает. Если сломалась — нашёл регрессию.
- **z-index гонка:** убедись что mobile-menu НЕ выше навигации с бургером. Иначе крестик не кликнуть.
- **transform: translateX в `.sh-horizontal-track`:** при back-navigation transform может остаться "грязным". В JS galery init pop'ит `transform: translateX(0)` перед пересчётом? Если нет — добавь reset в `initHorizontalScroll`.
