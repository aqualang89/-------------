## Что менять

### Пункт 1: кнопка "Назад" со страницы товара → в каталог

Файл: `layouts/default.vue`

В `<script setup>` добавь computed:
```js
const route = useRoute()
const backTo = computed(() => {
  if (route.path.startsWith('/catalog/') && route.path !== '/catalog') return '/catalog'
  return '/'
})
```

В template замени `to="/"` на `:to="backTo"`:
```vue
<NuxtLink v-if="$route.path !== '/'" :to="backTo" class="sh-back" @click.native="sessionStorage.removeItem('introShown')">
  ← Назад
</NuxtLink>
```

**Проверка:**
- На `/catalog/aquarium-200l` (страница товара) → кнопка ведёт на `/catalog`
- На `/catalog` (сам каталог) → кнопка ведёт на `/`
- На `/services`, `/about`, `/calculator`, `/cart` → кнопка ведёт на `/`
- На `/` (главная) → кнопки нет (она и так скрыта через `v-if`)

### Пункт 2: tap-area кнопки "Назад" на мобиле

Файл: `layouts/default.vue`, мобильный медиа `.sh-back`

Текущее (строки 319–325):
```css
@media (max-width: 768px) {
  .sh-back {
    position: static;
    margin: 16px 20px 0;
    display: inline-block;
  }
}
```

Сделать:
```css
@media (max-width: 768px) {
  .sh-back {
    position: static;
    margin: 8px 12px 0;
    display: inline-flex;
    align-items: center;
    padding: 12px 16px;
    min-height: 44px;
    font-size: 12px;
    -webkit-tap-highlight-color: rgba(217, 180, 106, 0.15);
  }
}
```

**Что меняется:** добавляем `padding: 12px 16px` + `min-height: 44px` (Apple/Material минимум для tap), `display: inline-flex` чтобы padding работал правильно, `font-size: 12px` (с 11px — чуть крупнее, читаемее), tap highlight для визуального feedback на Android. Margin уменьшили чтобы не было лишнего пустого пространства.

### Пункт 3: nav прижата к правому краю на широких мониторах

Файл: `layouts/default.vue`, селектор `.sh-nav-inner` (строки 100–108)

Текущее:
```css
.sh-nav-inner {
  max-width: 1440px;
  margin: 0 auto;
  display: flex;
  justify-content: flex-end;
  align-items: flex-start;
  gap: 28px;
  pointer-events: auto;
}
```

**Простое решение:** убрать `max-width` совсем. `padding: 22px 48px` у `.sh-nav` уже задаёт отступ от краёв экрана. Без `max-width` nav-inner будет растягиваться на всю ширину между padding'ами, и `justify-content: flex-end` прижмёт ссылки/бургер к правому краю padding'а (48px от края экрана) на любой ширине.

Замена:
```css
.sh-nav-inner {
  display: flex;
  justify-content: flex-end;
  align-items: flex-start;
  gap: 28px;
  pointer-events: auto;
}
```

(убрали `max-width: 1440px` и `margin: 0 auto`)

**Проверка:**
- На мониторе 1920px — ссылки в 48px от правого края экрана
- На 2560px — то же, в 48px от правого края
- На 1440px и ниже — без изменений (там max-width не срабатывал)
- На мобиле — мобильный медиа `padding: 12px 20px` остаётся

### Пункт 4: градиент сверху картинки аквариума на десктопе

Файл: `pages/index.vue`, селектор `.sh-interior`

Сейчас на десктопе у `.sh-interior` нет своих `::before / ::after` (они только в мобильном медиа на строках 1158+). На десктопе работают `.sh-stage::before` (сверху) и `.sh-stage::after` (снизу) — но они на уровне `.sh-stage`, не самой картинки, и переход от фона страницы к картинке остаётся резким сверху.

**Решение:** добавить десктопный `.sh-interior::before` с градиентом сверху. По аналогии с мобильным, но десктопно — высота побольше (например, 25% высоты картинки или 200px).

После блока `.sh-interior { ... }` (строки 496-508) добавь:
```css
.sh-interior::before {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  height: 180px;
  background: linear-gradient(to bottom, #1e2933 0%, rgba(30, 41, 51, 0.5) 50%, transparent 100%);
  pointer-events: none;
  z-index: 3;
}
```

`z-index: 3` — выше самой картинки (`.sh-interior` имеет `z-index: 1`), но не конфликтует с `.sh-stage::before/after` (у них `z-index: 2`). На мобиле этот `::before` перезапишется существующим в мобильном медиа.

**Проверка:**
- На десктопе верх картинки аквариума плавно растворяется в фон страницы (нет резкой линии)
- Нижний градиент (`.sh-stage::after`) не сломан, переход снизу остаётся плавным
- На мобиле — без регрессий, там свой `::before` через медиа
