## Архитектура Excel-импорта (ФИНАЛЬНАЯ)

### Две таблицы

| Таблица | Источник | Содержимое | Частота |
|---------|----------|-----------|---------|
| **Справочник** | `555-with-articles.xlsx` (ручной) | Артикул, название, группа, подгруппа, цена, остаток | Редко — при новых товарах |
| **Остатки 1С** | `333.xlsx` (выгрузка 1С) | Название, цена, остаток | Каждый день |

### Алгоритм загрузки

```
1. Загружаем справочник 555.xlsx → таблица products (article, name, category_id, price)
   - Парсим дерево категорий из цепочки заголовков
   - Маппим на эталонное дерево (скрин заказчика)
   - Batch upsert по артикулу

2. Загружаем остатки из 1С 333.xlsx (каждый день):
   Для каждого товара:
   ├── Ищем по названию (строгое совпадение ===) в products
   │     ├── НАШЛИ → обновляем цену, остаток, is_available = true
   │     └── НЕ НАШЛИ → СОЗДАЁМ новый товар:
   │           article = генерируем автоматически (NEW-NNN)
   │           category_id = "НОВИНКИ"
   │           is_new = true
   │           (клиент потом разбирает через админку)
   │
   └── Товары из products, которых нет в 1С
         └── is_available = false (нет в наличии)
```

### Генерация артикулов
- Безартикульные товары из справочника → `SH-NNN` (Scaper's House)
- Новые товары из 1С → `NEW-NNN`
- Фото называются по артикулу (`SH-001.jpg`, `Den-3929.jpg`)

### Обработка новых товаров
- Новый товар в 1С, которого нет в справочнике → группа "НОВИНКИ"
- Клиент через админку меняет группу при необходимости
- Не требуется ручное вмешательство в загрузку

### Исчезнувшие товары
- Если товар есть в справочнике, но не пришёл в 1С → `is_available = false`
- На сайте отображается как "нет в наличии"

## Правила сопоставления
- **Строгое совпадение** по названию (без fuzzy matching)
- Названия в 1С не меняются (гарантия клиента)
- Два товара с одинаковым названием — надеемся, не будет

## Фото
- Фотограф называет файлы по артикулу (SH-001.jpg, Den-3929.jpg)
- Загрузка фото на сайт → автопривязка к товару по артикулу
